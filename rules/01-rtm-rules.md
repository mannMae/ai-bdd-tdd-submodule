# 1. 기술 매핑 및 자가 채점표 (Technical RTM) 원칙

모든 프로젝트의 세부 기능 구현 정보와 규칙 준수 현황은 기능별 기술 매핑 문서(`rtm_{기능명}.md`)에서 관리하며, 다음의 원칙을 따릅니다.

## 1) 설계 및 작성 원칙
1. **세부 컬럼 단위 매핑**: 각 시나리오별로 프론트엔드(UI, API, 상태/스토어, 브라우저/유틸), 백엔드(API, 아키텍처/인프라, DB 테이블, 도메인), AI 모듈(API/입출력, 모델/설정), 글로벌 아키텍처의 연관 정보를 세부 컬럼으로 전개하여 매핑하며, 해당없음/연동없음은 반드시 `-`로 명시합니다.
   - **영역별 시각적 구분선 (┃)**: 각 프로젝트 영역(FE, BE, AI, 글로벌)의 명확한 시각적 구분을 위해, 테이블 사이 경계에 굵은 세로 실선(`┃`) 문자 컬럼을 삽입하여 구획을 나눕니다.
   - **[FE] UI**: 시나리오 수행 시 활성화되는 대상 UI 흐름을 기재합니다. 화면 이동 시 `이전UI ➔ 이동할UI` 형태로 변화를 명시하며, 중간에 등장하는 팝업/모달/서브패널 등은 괄호 `(SubUI)`를 사용하여 흐름 사이에 표시합니다. UI가 개입하지 않는 경우 `-`로 기재합니다.
   - **[FE] 상태/스토어**: Zustand 등 상태 관리 스토어 매핑 정보를 기재합니다.
   - **[FE] 브라우저/유틸**: LocalStorage, SessionStorage, Cookie 등 브라우저 저장소 연동 정보나 Native API 활용 정보, 혹은 연산/파싱용 공통/도메인 유틸리티 함수(utils)의 경로를 기재합니다.
   - **[BE] API (HTTP/WS)**: 백엔드가 외부 요청을 받아들이는 라우터 인터페이스 정보(HTTP 경로, WebSocket 이벤트 등)를 기재합니다.
   - **[BE] 아키텍처/인프라**: 비동기/동시성 제어, 메시지 브로커, 백그라운드 큐 등 시스템을 구동하는 미들웨어/프레임워크 연동 정보(Celery, Redis Pub/Sub 메시지 발행/수집 등)를 기재합니다.
   - **[BE] DB 테이블**: 해당 시나리오에서 데이터가 영속적으로 저장, 수정, 또는 조회되는 데이터베이스 테이블명이나 ORM 모델명을 기재합니다.
   - **[BE] 도메인 (Usecase/VO)**: 비즈니스 연산 단위인 Usecase 클래스명과 값 객체인 VO를 한 칸에 묶어서 기재합니다.
   - **[AI] API/입출력**: AI 모듈/엔진 서버가 제공하는 추론 API와 입출력 명세(입력 신호 형태 및 최종 결과 필드 등)를 기재합니다.
   - **[AI] 모델/설정**: 추론에 사용되는 머신러닝/인공지능 모델 형식(ONNX 등) 및 가중치 정보와 모델 구동 설정(Outlier Filter 온오프, 정규화 등)을 기재합니다.
2. **구체적 증거 기반 자가 채점**: 구현 완료 후, 코드가 개발 가이드레일 규칙을 준수했는지 스스로 채점(Pass/Fail)해야 하며, 대충 통과 표시를 하지 말고 증거가 되는 구체적인 코드 파일 경로 및 라인 링크를 반드시 제시해야 합니다.
3. **검증 경로 일원화**: 문서 하단에 해당 시나리오를 검증하기 위한 거킨 파일, 테스트 파일, 구현 소스 코드 파일의 실제 경로 링크를 등록하여 검증 경로를 한눈에 볼 수 있게 일원화합니다.

## 2) 운영 및 검증 규칙 (중요)
1. **구현 완료 시 행 업데이트**: 각 기능(시나리오)의 구현 및 테스트 통과가 완료되면, 에이전트는 즉시 해당 시나리오 행(Row)의 상태를 업데이트하고 자가 채점표를 채워야 합니다.
2. **아키텍처 변경 시 전수 조사 및 동기화**: 글로벌 아키텍처나 코어 인프라 규칙이 업데이트/변경되면, 에이전트는 작성된 모든 `rtm_*.md` 문서를 전수 조사하여 새로운 아키텍처의 적용 필요 여부를 식별하고, 해당되는 매핑 정보와 자가 채점표를 즉시 최신화해야 합니다.
3. **셀(Cell) 단위 테스트 검증**: 매트릭스 표에 명시된 프론트 컴포넌트, 백엔드 API, 아키텍처 등의 각 셀 단위 기술 요소들은 반드시 독립적으로 검증될 수 있는 단위 테스트(Unit Test) 혹은 통합 테스트 코드가 동반되어야 합니다.
